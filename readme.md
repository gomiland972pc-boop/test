# MAX-BOT

Бот поддержки для МАКС (Разработка происходила на новой версии Python 3.14.4).
Работает через **webhook** (`POST /subscriptions` в Bot API МАКС) — long polling в новой версии API отключён.

## Установка

### Linux/macOS (рекомендуется через venv)

```
python3 -m venv .venv
./.venv/bin/pip install -r requirements.txt
```

На Debian/Ubuntu системный pip заблокирован (PEP 668), поэтому venv обязателен.
Если `python3 -m venv` ругается на отсутствие модуля — установите `python3-venv`:
`sudo apt install python3-venv`.

### Windows (PowerShell)

```
python -m venv .venv
.\.venv\Scripts\pip.exe install -r requirements.txt
```

## Настройка

Скопируйте шаблон и заполните значения:

```
cp .env.example .env
```

Затем откройте `.env` и впишите как минимум `MAX_BOT_TOKEN`, `ADMIN_IDS` и `WEBHOOK_URL`.

```
MAX_BOT_TOKEN=ваш_токен
ADMIN_IDS=6228168
DB_PATH=bot.db
MAX_API_BASE=https://botapi.max.ru

# Webhook
WEBHOOK_URL=https://your-domain.com/webhook
WEBHOOK_SECRET=любая_строка_5_256_символов_a-zA-Z0-9_-
WEBHOOK_PATH=/webhook
WEBHOOK_HOST=0.0.0.0
WEBHOOK_PORT=8080
```

`ADMIN_IDS` можно указать через запятую:

```
ADMIN_IDS=111111,222222,333333
```

### Требования к URL webhook (из документации МАКС)

- Только `https://`, порт 443 (в URL не указывается).
- Действительный сертификат от доверенного CA (самоподписанные не работают).
- Endpoint должен ответить `HTTP 200` в течение 30 секунд.
- Если задан `WEBHOOK_SECRET`, МАКС передаёт его в заголовке `X-Max-Bot-Api-Secret` — бот проверяет совпадение.

`WEBHOOK_HOST`/`WEBHOOK_PORT` — это локальные адрес и порт, на которых поднимается aiohttp-сервер.
`WEBHOOK_URL` — публичный HTTPS-адрес, по которому МАКС достучится до этого сервера
(обычно через reverse-proxy вроде nginx / caddy с TLS-сертификатом).

## Запуск

Из папки проекта:

```
./.venv/bin/python main.py     # Linux/macOS
.\.venv\Scripts\python.exe main.py    # Windows PowerShell
```

При старте бот:
1. Поднимает HTTP-сервер на `WEBHOOK_HOST:WEBHOOK_PORT`.
2. Удаляет старую подписку (если была) и регистрирует новую через `POST /subscriptions`.
3. Принимает обновления на `POST {WEBHOOK_PATH}` и отвечает `200` сразу,
   обрабатывая update в фоне.

Эндпоинт `GET /healthz` возвращает `ok` для проверки живости.

## Команды

/start — запуск бота и главное меню
/menu — открыть главное меню
/admin — открыть список тикетов для администраторов

## Важно

Обычные пользователи не имеют доступа к админ-панели.

## Что нового

- **Авто-статус «На рассмотрении»**. При создании тикета и после каждого сообщения
  пользователя бот автоматически переводит тикет в статус «На рассмотрении» —
  но только если админ не выставлял статус вручную. Как только админ ставит
  «Изучает специалист» / «Готовится ответ» / «Закрыт», авто-статус больше не
  переключается.
- **«Написать пользователю»** в админ-панели. Кнопка `✍️ Написать пользователю`
  открывает диалог: админ вводит ID, затем первое сообщение (можно с медиа).
  Создаётся новый тикет с порядковым номером от «Службы поддержки» —
  у пользователя он отображается в активных, у админа — так же, как обычные тикеты.
- **Новый формат истории тикета**: От кого / Кому / ID / Статус / Создан / Обновлён.
- **Вложения** (фото/файлы/видео) теперь сохраняются в истории сообщений и
  пересылаются между пользователем и админом «как есть».
